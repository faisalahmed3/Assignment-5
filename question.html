Question-2: What is the difference between innerHTML, innerText, and textContent?

The difference between "innerHTML", "innerText", and "textContent" lies in how they handle text and HTML. "innerHTML" retrieves or sets HTML content, including tags, making it useful for dynamic updates but risky for XSS attacks. "innerText" only gets or sets visible text, ignoring hidden elements and formatting. "textContent" retrieves all text, including hidden content, but treats everything as plain text, making it faster than "innerHTML". Use "innerHTML" for handling HTML, "innerText" for visible content, and "textContent" for efficient text extraction.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a technique where a single event listener is added to a parent element instead of multiple child elements. This works due to event bubbling, where an event triggered on a child propagates up to its ancestors. Using "event.target", we can identify which child element triggered the event and handle it accordingly. This approach improves performance by reducing the number of event listeners, simplifies code maintenance, and allows handling dynamically added elements without manually attaching new event listeners. It is commonly used in scenarios like handling clicks on lists, forms, or dynamically loaded content efficiently.

Question-4: What is event bubbling in the DOM?

Event bubbling in the DOM is when an event on a child element propagates up through its parent elements. For example, a click on a button inside a "div" triggers the event on the button first, then on the "div", and continues up to "body" and beyond. This allows event delegation but can cause unintended behavior if multiple elements handle the same event. Bubbling can be stopped using `event.stopPropagation()`.

Question-5: How do you create, add, and remove elements using JavaScript?

In JavaScript, you can create, add, and remove elements dynamically using various methods. To create an element, use "document.createElement()", which generates a new element that can be modified before adding it to the DOM. Adding an element can be done with methods like "appendChild()", which places the element as the last child,"prepend()", which inserts it as the first child, and "insertBefore()", which places it before a specified element. To remove elements, "removeChild()" can be used to delete a child from its parent, while "remove()" directly removes an element from the DOM. These methods allow efficient manipulation of web pages, making them interactive and dynamic.
